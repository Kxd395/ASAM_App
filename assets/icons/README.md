## Design Specifications

### Color Palette

- **Primary**: Slate Blue `#2A63A5` (professional, healthcare-appropriate)
- **Secondary**: Light Gray `#F4F6F8` (clean background)
- **Accent**: Coral `#FF7A5A` (warmth, engagement)

### Motif

- Circular wheel with six equal segments
- Each segment represents one ASAM dimension (D1-D6)
- Minimal, modern aesthetic
- Safe zone aware for adaptive icons

### Accessibility

- Contrast ratio: 4.5:1 (WCAG AA compliant)
- No fine detail or text
- Clear at small sizes (29px+)
- Central mark stays inside safe circle

## Platform-Specific Rules

### iOS / macOS
- **App Store listing**: 1024x1024 PNG, no alpha, no rounded corners
- **Binary icons**: All sizes from asset catalog
- **Format**: PNG, sRGB color space
- **Corners**: System applies mask, do NOT add rounded corners to artwork

### Android / Google Play
- **Play Store listing**: 512x512 PNG
- **Launcher**: Adaptive icons (separate foreground + background layers)
- **Legacy**: 192x192 and 512x512 for older devices
- **Safe zone**: Keep primary mark inside central 66% circle

### Export Commands

**PNG exports from SVG** (using Inkscape or ImageMagick):
```bash
# iOS App Store
inkscape icon_master.svg --export-filename=AppStore-1024x1024.png --export-width=1024 --export-height=1024

# All iOS sizes
for size in 40 60 58 87 80 120 180 152 167; do
  inkscape icon_master.svg --export-filename=AppIcon-${size}.png --export-width=${size} --export-height=${size}
done

# Android Play Store
inkscape icon_master.svg --export-filename=play_store_512x512.png --export-width=512 --export-height=512

# Android adaptive layers
inkscape icon_master.svg --export-filename=adaptive_foreground.png --export-width=1080 --export-height=1080
# (Create solid background layer separately)
```

**macOS icns** (using iconutil):
```bash
# Generate all sizes into .iconset
iconutil -c icns AppIcon.iconset -o AppIcon.icns
```

## Integration Steps

### 1. Add to Xcode

1. Open `ASAMAssessment.xcodeproj`
2. Select `Assets.xcassets` in Project Navigator
3. Right-click → New iOS App Icon
4. Drag all PNG files from `ios/AppIcon.appiconset/` into corresponding slots
5. Update `Info.plist`:
   ```xml
   <key>CFBundleDisplayName</key>
   <string>CarePath Assess</string>
   <key>CFBundleIdentifier</key>
   <string>com.carepath.assess</string>
   ```

### 2. Add to Android (if applicable)

1. Copy adaptive layers to `res/mipmap-xxxhdpi/`
2. Create `res/mipmap-anydpi-v26/ic_launcher.xml`:
   ```xml
   <?xml version="1.0" encoding="utf-8"?>
   <adaptive-icon xmlns:android="http://schemas.android.com/apk/res/android">
       <background android:drawable="@mipmap/adaptive_background"/>
       <foreground android:drawable="@mipmap/adaptive_foreground"/>
   </adaptive-icon>
   ```
3. Update `AndroidManifest.xml`:
   ```xml
   <application android:label="CarePath Assess" ...>
   ```
